## What is this?

[NeetCode.io](https://neetcode.io) is a coding interview preparation platform featuring curated problems, video solutions, and an in-browser code editor. This repository is automatically populated with your accepted (or all) solutions using the **GitHub Sync** feature.

## How GitHub Sync works

1. **Connect your GitHub account** on [neetcode.io/profile/github](https://neetcode.io/profile/github).
2. **Auto-commit** ŌĆö every time you submit a solution on NeetCode, it is pushed here automatically (configurable by status).
3. **Bulk Sync** ŌĆö push all your past solutions at once from the GitHub settings page.
4. **Manual sync** ŌĆö from the submission history panel on any problem page, sync or remove individual submissions.

## Algostudy archive sync

This repository also feeds [`samu9nai/algostudy`](https://github.com/samu9nai/algostudy) through `.github/workflows/sync-to-algostudy.yml`.

- NeetCode submissions under `Data Structures & Algorithms/` are normalized into `algostudy/LeetCode/`.
- LeetCode-only submissions can be added manually under `Only in LeetCode/<number>-<slug>/`.
- NeetCode and LeetCode slug mismatches are resolved in `scripts/problem-map.json`.
- If the `GEMINI_API_KEY` repository secret is present, the README analysis sections are generated with Gemini using `GEMINI_MODEL` (`gemini-3.1-flash-lite` by default).
- Generated analysis is cached with an invisible README marker and reused until the problem statement or solution code changes.
- If Gemini is unavailable, the script falls back to deterministic local analysis so the sync still succeeds.
